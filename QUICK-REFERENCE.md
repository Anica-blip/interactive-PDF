### Step 3: Add Interactive Elements

```
┌─────────────────────────────────────────────────────────┐
│  ADD VIDEO                                              │
└─────────────────────────────────────────────────────────┘

1. Click "Add Video" button
2. Upload video file (uploads to Cloudflare Stream)
3. Drag video hotspot to position
4. Resize to match thumbnail: 500 × 375

   ┌─────────────────────────────────┐
   │  [PNG Background with thumbnail]│
   │  ┌─────────────────────────┐   │
   │  │ [Video Hotspot]         │   │
   │  │  Drag corners to resize │   │
   │  │  500 × 375              │   │
   │  └─────────────────────────┘   │
   └─────────────────────────────────┘

Result: User clicks thumbnail → Video plays in floating player!

┌─────────────────────────────────────────────────────────┐
│  ADD BUTTON                                             │
└─────────────────────────────────────────────────────────┘

1. Click "Add Button" button
2. Enter URL (internal or external)
3. Drag to position
4. Resize: 200 × 60

┌─────────────────────────────────────────────────────────┐
│  ADD AUDIO                                              │
└─────────────────────────────────────────────────────────┘

1. Click "Add Audio" button
2. Upload audio file (uploads to R2)
3. Drag to position
4. Resize: 400 × 60 (wide player bar)

┌─────────────────────────────────────────────────────────┐
│  ADD GIF                                                │
└─────────────────────────────────────────────────────────┘

1. Click "Add GIF/Image" button
2. Upload GIF file (uploads to R2)
3. Drag to position
4. Resize as needed (maintains aspect ratio)
```

## 🎯 Video Thumbnail Strategy

### ✅ RECOMMENDED: Add thumbnail in Canva

```
WHY IT WORKS:
┌─────────────────────────────────────────────────────────┐
│  1. User Experience                                     │
│     • Sees thumbnail immediately (no loading)           │
│     • Knows what to expect                              │
│     • Professional appearance                           │
│                                                         │
│  2. Performance                                         │
│     • Page loads fast                                   │
│     • Video loads only when clicked                     │
│     • Bandwidth efficient                               │
│                                                         │
│  3. Fallback                                            │
│     • If video fails, thumbnail still visible           │
│     • Better than blank space                           │
│     • User knows content exists                         │
└─────────────────────────────────────────────────────────┘

HOW TO DO IT:
┌─────────────────────────────────────────────────────────┐
│  In Canva:                                              │
│  1. Get video thumbnail:                                │
│     • Screenshot from video                             │
│     • OR use Cloudflare Stream auto-thumbnail          │
│     • OR create custom graphic                          │
│                                                         │
│  2. Add to design:                                      │
│     • Insert as image                                   │
│     • Size: 500 × 375 px                                │
│     • Optional: Add ▶ play button overlay              │
│     • Optional: Add border/shadow                       │
│                                                         │
│  3. Export as PNG                                       │
│                                                         │
│  In Builder:                                            │
│  1. Upload PNG (with thumbnail)                         │
│  2. Add video hotspot                                   │
│  3. Position EXACTLY over thumbnail                     │
│  4. Resize to match thumbnail size                      │
│                                                         │
│  Result:                                                │
│  User clicks thumbnail → Video plays in floating player!│
└─────────────────────────────────────────────────────────┘
```

## 🚀 Vercel → Cloudflare Migration

### The Problem:
```
Old URLs: https://your-app.vercel.app/pdfs/file.pdf
New URLs: https://files.3c-public-library.org/pdfs/file.pdf
Issue:    Database has old URLs, slow uploads
```

### ✅ The Solution:
```
1. Deploy vercel.json (automatic redirects)
   • Old URLs redirect to Cloudflare
   • No database changes needed
   • Zero downtime

2. Upload directly to R2 (faster)
   • Use Worker API
   • Bypass Supabase
   • Instant uploads

3. Update database (optional, later)
   • Run SQL replacement script
   • Clean up URLs
   • When you have time
```

### Quick Fix (5 minutes):
```bash
# Already done! Just deploy:
cd /home/acer/CascadeProjects/interactive-PDF-main
git push

# Vercel auto-deploys
# Redirects active in ~2 minutes
# All old links work!
```
